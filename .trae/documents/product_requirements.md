# 产品需求文档 (PRD) - 名人消息实时聚合助手

## 1. 项目概述
### 1.1 产品背景
用户需要一个能够实时获取特定关注人物（如特朗普、马斯克、李笑来）在全网最新动态的工具。该工具需要跨越单一社交平台的限制，聚合来自 X (Twitter)、微博、微信公众号、Instagram、Reddit 等多个来源的信息，并利用 AI 进行筛选和摘要，提供高效的信息获取体验。

### 1.2 核心价值
- **跨平台聚合**：一站式查看人物在不同社交媒体的动态，无需切换多个 App。
- **即时性**：获取 48 小时内的最新消息，确保信息鲜度。
- **AI 智能摘要**：利用大模型过滤无关噪音，只看核心内容。
- **专属服务**：个人私有化部署，数据隐私可控。

## 2. 功能需求

### 2.1 核心功能
| 功能模块 | 优先级 | 功能描述 |
| :--- | :--- | :--- |
| **人物关注管理** | P0 | 系统预设关注（特朗普、马斯克、李笑来）；用户可查看关注列表。 |
| **人物搜索与确认** | P1 | 用户输入人名 -> 系统调用搜索 API -> 返回人物简介/头像 -> 用户确认是否为目标人物 -> 加入关注列表。 |
| **消息流聚合** | P0 | 展示已关注人物的最新动态卡片。每条消息包含：来源图标、发布时间、摘要内容、原文链接。 |
| **一键刷新** | P0 | 提供全局/单人刷新按钮。点击后实时调用 API 获取最新数据。 |
| **访问控制** | P0 | **新增**：简单的访问码（Access Code）机制。首次访问需输入密码，防止他人盗用 API 额度。 |

### 2.2 用户界面 (UI)
- **首页/仪表盘**：顶部为状态栏（刷新按钮、最后更新时间），主体为关注人物的消息流（Timeline 形式或分栏形式）。
- **添加人物弹窗**：搜索框 -> 结果预览 -> 确认按钮。
- **登录页**：简单的输入框，验证 Access Code。

## 3. 非功能需求
- **响应速度**：搜索和刷新操作需有明确的 Loading 状态（骨架屏）。
- **移动端适配**：需支持手机浏览器完美显示。
- **成本控制**：优先使用免费/低成本 API 方案。

## 4. 载体选择分析
**结论：选择 Web 网页 (Next.js 应用)**
- **原因 1：开发维护成本低**。相比浏览器插件需要适配 Chrome/Edge/Firefox 不同标准及审核流程，Web 应用只需部署一次。
- **原因 2：移动端可访问**。浏览器插件在手机上体验极差或无法使用，Web 网页随时随地可开。
- **原因 3：后端能力**。聚合搜索和 LLM 处理需要服务端环境来隐藏 API Key，纯前端插件安全性较低。
